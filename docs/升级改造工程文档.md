### 4.2 Pomelo 说明

**Pomelo.EntityFrameworkCore.MySql** 是 EF Core 的 MySQL 数据库提供程序：

```
EF Core（ORM框架）
    │
    ├── Microsoft.EntityFrameworkCore.SqlServer  → SQL Server
    ├── Microsoft.EntityFrameworkCore.Sqlite     → SQLite（您之前用的）
    └── Pomelo.EntityFrameworkCore.MySql         → MySQL（新项目用的）
```

只需要换一个 NuGet 包，代码几乎不用改！

### 4.5 从 SQLite 切换到 MySQL

只需修改配置：

```csharp
// 旧项目（SQLite）
services.AddDbContext<AppDbContext>(options =>
    options.UseSqlite(connectionString));

// 新项目（MySQL）
services.AddDbContext<AppDbContext>(options =>
    options.UseMySql(connectionString, 
        ServerVersion.AutoDetect(connectionString)));
```

## 五、Git 使用指南

### 5.1 Git vs SVN 对比

| 概念 | SVN | Git |
|------|-----|-----|
| 提交 | `svn commit` | `git commit` + `git push` |
| 更新 | `svn update` | `git pull` |
| 检出 | `svn checkout` | `git clone` |
| 分支 | 服务器端创建，重量级 | 本地创建，轻量级 |
| 仓库 | 集中式（服务器） | 分布式（本地+远程） |

### 5.2 Git 常用命令

#### 初始化和克隆
```bash
git init                    # 初始化本地仓库
git clone <url>             # 克隆远程仓库
```

#### 日常操作（最常用）
```bash
git status                  # 查看状态
git add .                   # 添加所有文件到暂存区
git commit -m "提交说明"    # 提交到本地仓库
git push                    # 推送到远程仓库
git pull                    # 拉取远程更新
```

#### 分支操作
```bash
git branch                  # 查看分支
git branch dev              # 创建 dev 分支
git checkout dev            # 切换到 dev 分支
git checkout -b feature/xx  # 创建并切换（合二为一）
git merge dev               # 合并 dev 到当前分支
git branch -d dev           # 删除分支
```

#### 查看历史
```bash
git log                     # 查看提交历史
git log --oneline           # 简洁模式
git diff                    # 查看修改内容
```

### 6.1 MySQL 表结构

#### 6.1.1 车辆表 (car_info)
```sql
CREATE TABLE car_info (
    car_id INT AUTO_INCREMENT PRIMARY KEY,

    owner_name VARCHAR(50) COMMENT '车主姓名',
    phone_number VARCHAR(50) COMMENT '电话号码',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT='车辆信息表';
```

#### 6.1.2 地点表 (address_info)

#### 6.1.5 业务记录表 (business_record)
```sql
CREATE TABLE business_record (
    business_id VARCHAR(15) PRIMARY KEY COMMENT '业务ID',
    car_id INT NOT NULL COMMENT '车辆ID',
    license_plate VARCHAR(200) NOT NULL COMMENT '车牌号',
    business_date DATE COMMENT '业务日期',
    start_addr_id INT COMMENT '起始地点ID',
    start_addr VARCHAR(200) COMMENT '起始地点',
    end_addr_id INT COMMENT '终止地点ID',
    end_addr VARCHAR(200) COMMENT '终止地点',
    material_id INT COMMENT '物料ID',
    material_name VARCHAR(200) COMMENT '物料名称',
    project_id INT COMMENT '项目ID',
    project_name VARCHAR(200) COMMENT '项目名称',
    car_count INT DEFAULT 0 COMMENT '车数',
    price DECIMAL(12,2) DEFAULT 0 COMMENT '单价',
    total_amount DECIMAL(12,2) DEFAULT 0 COMMENT '总金额',
    is_paid TINYINT DEFAULT 0 COMMENT '是否付款(0:未付,1:已付)',
    is_cancelled TINYINT DEFAULT 0 COMMENT '是否作废(0:正常,1:作废)',
    memo VARCHAR(200) COMMENT '备注',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_business_date (business_date),
    INDEX idx_license_plate (license_plate)
```

### 6.2 数据迁移方案

从 SQL Server 迁移到 MySQL 的步骤：

1. **导出 SQL Server 数据**
   - 使用 SQL Server Management Studio 导出为 CSV
   - 或使用迁移工具（如 MySQL Workbench Migration）

2. **数据转换**
   - 字段名称转换（驼峰 → 下划线）
   - 数据类型转换
   - 日期格式转换

3. **导入 MySQL**
   - 使用 LOAD DATA INFILE 或 INSERT 语句

4. **数据验证**
   - 检查记录数量
   - 检查数据完整性
